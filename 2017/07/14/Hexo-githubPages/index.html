Hexo+GithubPages搭建个人Bolg分享

2017-07-14 • 分类: 技术分享 • 标签: 技术分享#文章標籤 可以省略

简单的说
1.Hexo相当于一家装修公司
2.GitHubPages相当于一个”免费”房子

你可以先去选择装修风格，购买家具，入住的时间自己选择。也可以先购买房子，然后再去挑选装修公司。
任君选择，下面分四部分讲：

一、前提:
Node.js （我理解为用终端命令就能下载软件的环境）
下载地址：https://nodejs.org/en/download/

Git（我理解为房子的开发商物业）
下载地址：https://git-scm.com/download/

下载安装后，在终端中检查一下是否安装好了，验证方法就是检查你本机安装这两个环境的版本号
node –version
npm –version
git –version

二、Hexo篇:
1.安装Hexo 先创建一个文件夹，然后cd进去
以下全部都是终端命令完成除了最后一步在浏览器中预览

2.npm install hexo-cli -g

3.npm install hexo –save

4.检查版本是否安装成功：hexo –version

5.hexo init

6.npm install

7.hexo g （hexo generate生成，一般是在本地生成了新的文件需要同步的时候）

8.hexo s （hexo server启动服务预览，每一次浏览本地Bolg的样子需要输入，停止是Ctrl + C）

9.在浏览器中打开 http://localhost:4000/ （浏览本地生成的样子 这个主题是默认的主题）

三、Github:
1.https://github.com

2.创建-登陆

3.直接右上角➕

4.New repository

5.Repository name格式:
你自己的用户名.github.io

下面是终端操作：
6.在本地创建一个文件夹来管理github.io cd到新建文件夹中

下载你的github.io：
clone https://github.com/你自己的用户名/你自己的用户名.github.io

四、把本地Hexo同步到Github:

1.敲黑板：你会碰见两个_config.yml
第1个应该会在你创建hexo文件夹中 这个主要负责博客主题的名字 描述 页数 我感觉就像硬件设施吧

第2个在hexo里面的themes的默认主题landscape文件夹中 如果你需要更改主题的 菜单名字 菜单的地址指向本地的哪个文件夹

关联hexo和githubpages 需要在第一个硬件设施的_config.yml，可以用记事本打开，但是如果windows电脑保存的时候记得选择格式UTF-8，因为中文会乱码。
找到：
deploy:
type: git
repo: git@github.com:你的用户名/你的用户名.github.io.git
branch: master
(切记：所有都是英文的，英文冒号后面必须跟着一个空格)
如果你更换主题：也是在该硬件_config.yml设施设置 theme:landscape （默认主题）

在终端输入 cd进入你hexo的文件夹
hexo g
hexo s
hexo d （部署）

大功告成～！！！

然后终端cd到你自己的用户名.github.io 文件夹下
git pull（更新成最新的）

五、绑定域名
1.购买 我在阿里云买的
2.在终端cd到你自己的用户名.github.io 文件夹下 创建一个关联域名的CNAME
创建方法：
touch CNAME
vim CNAME (打开 i是编辑 输入你的域名 不用加wwww 退出编辑是esc然后 :wq 回车)
git add CNAME
git commit -m “add CNAME” (-m 的意思是添加注释)

然后登陆阿里云解析一下你的域名
在终端输入 ping https://github.com/你自己的用户名/你自己的用户名.github.io
查一下ip地址添加解析会用到
添加三个解析
1.记录类型：A 主机记录：WWW 记录值：你的IP地址
2.记录类型：A 主机记录：@ 记录值：你的IP地址
3.记录类型：CNAME 主机记录：blog 记录值：你自己的用户名.github.io

六、编辑调试本地的bolg，然后不断的
hexo g
hexo s
hexo d （部署）

git 下记得 gitpull -> git push

编辑中遇到的坑:
1.修改主题的菜单，怎么也修改不了，后来发现是在第一个_config.yml应该在主题里面的_config.yml修改

menu:
首页: /#blog
简介: /about
分类: /categories
（／后面接的是你themes文件夹并列的source中地址）

2.创建的菜单文章都在和themes文件夹并列的source中
_posts文件夹下面是文章
categories文件及下面是对应的菜单
about文件及下面是对应的菜单

我的categories中打开index.md
title: 分类
date: 2017-07-14 14:03:54
layout: page-archive
是这个样子的，是希望layout一个负责分类的菜单

文章中需要引用图片是把图片也是放在source中
文章中直接
